# 为什么我们在服务端处理路径的时候一定要用path

因为服务端代码要运行在不同操作系统上（Windows、macOS、Linux），而它们的路径规则并不一致：

- Windows 常见分隔符是 `\`
- macOS / Linux 常见分隔符是 `/`
- 绝对路径格式、盘符规则、大小写敏感性也可能不同

如果我们手写字符串去拼路径（比如 `dir + "/" + file`），很容易在某些系统上出错，导致文件找不到、静态资源读取失败、构建产物路径异常等问题。

`path` 模块的核心价值是“跨平台统一”：

- `path.join()`：自动按当前系统规则拼接路径，避免重复斜杠和分隔符错误
- `path.resolve()`：把相对路径解析成绝对路径，减少运行目录变化带来的问题
- `path.dirname()`、`path.basename()`、`path.extname()`：稳定地解析路径信息
- `path.normalize()`：规范化路径，消除 `..`、`.` 和多余分隔符的影响

在构建工具（如 Vite / Webpack）或 Node 服务中，路径相关逻辑非常多。使用 `path` 不是“代码风格问题”，而是保证代码可移植性、稳定性和可维护性的基础做法。
